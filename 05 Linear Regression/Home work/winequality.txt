1. Название: Качество вина

2. Источники
    Создатели: Пауло Кортес (Университет Минью), Антонио Сердейра, Фернандо Алмейда, Тельмо Матос и Хосе Рейс (CVRVV) @ 2009 г.

3. Прошлое использование:

   П. Кортес, А. Сердейра, Ф. Алмейда, Т. Матос и Х. Рейс.
   Моделирование винных предпочтений путем интеллектуального анализа данных по физико-химическим свойствам.
   В системах поддержки принятия решений, Elsevier, 47(4):547-553. ISSN: 0167-9236.

   В приведенной выше ссылке были созданы два набора данных с использованием образцов красного и белого вина.
   Входные данные включают объективные тесты (например, значения pH), а выходные данные основаны на сенсорных данных.
   (медиана не менее 3 оценок, сделанных винными экспертами). Каждый эксперт оценил качество вина
   от 0 (очень плохо) до 10 (очень отлично). Для моделирования было применено несколько методов интеллектуального анализа данных.
   эти наборы данных с использованием регрессионного подхода. Модель машины опорных векторов достигла
   лучшие результаты. Было рассчитано несколько показателей: MAD, матрица неточностей для фиксированной толерантности к ошибкам (T),
   и т. д. Кроме того, мы строим график относительной важности входных переменных (измеряемых чувствительностью
   процедура анализа).

4. Соответствующая информация:

    Два набора данных относятся к красному и белому вариантам португальского вина «Виньо Верде».
    Для получения более подробной информации посетите: http://www.vinhoverde.pt/en/ или ссылку [Cortez et al., 2009].
    Из соображений конфиденциальности и логистики используются только физико-химические (входные данные) и сенсорные (выходные) переменные.
    доступны (например, отсутствуют данные о сортах винограда, марке вина, отпускной цене вина и т. д.).

    Эти наборы данных можно рассматривать как задачи классификации или регрессии.
    Классы упорядочены и не сбалансированы (например, нормальных вин гораздо больше, чем
    отличные или плохие). Алгоритмы обнаружения выбросов можно использовать для обнаружения нескольких отличных
    или плохие вина. Кроме того, мы не уверены, все ли входные переменные актуальны. Так
    было бы интересно протестировать методы выбора функций.

5. Количество экземпляров: красное вино – 1599; белое вино – 4898.

6. Количество атрибутов: 11 + выходной атрибут.

    Примечание: некоторые атрибуты могут быть коррелированы, поэтому имеет смысл применить какой-то тип
    выбор функции.

1. Title: Wine Quality 

2. Sources
   Created by: Paulo Cortez (Univ. Minho), Antonio Cerdeira, Fernando Almeida, Telmo Matos and Jose Reis (CVRVV) @ 2009
   
3. Past Usage:

  P. Cortez, A. Cerdeira, F. Almeida, T. Matos and J. Reis. 
  Modeling wine preferences by data mining from physicochemical properties.
  In Decision Support Systems, Elsevier, 47(4):547-553. ISSN: 0167-9236.

  In the above reference, two datasets were created, using red and white wine samples.
  The inputs include objective tests (e.g. PH values) and the output is based on sensory data
  (median of at least 3 evaluations made by wine experts). Each expert graded the wine quality 
  between 0 (very bad) and 10 (very excellent). Several data mining methods were applied to model
  these datasets under a regression approach. The support vector machine model achieved the
  best results. Several metrics were computed: MAD, confusion matrix for a fixed error tolerance (T),
  etc. Also, we plot the relative importances of the input variables (as measured by a sensitivity
  analysis procedure).
 
4. Relevant Information:

   The two datasets are related to red and white variants of the Portuguese "Vinho Verde" wine.
   For more details, consult: http://www.vinhoverde.pt/en/ or the reference [Cortez et al., 2009].
   Due to privacy and logistic issues, only physicochemical (inputs) and sensory (the output) variables 
   are available (e.g. there is no data about grape types, wine brand, wine selling price, etc.).

   These datasets can be viewed as classification or regression tasks.
   The classes are ordered and not balanced (e.g. there are munch more normal wines than
   excellent or poor ones). Outlier detection algorithms could be used to detect the few excellent
   or poor wines. Also, we are not sure if all input variables are relevant. So
   it could be interesting to test feature selection methods. 

5. Number of Instances: red wine - 1599; white wine - 4898. 

6. Number of Attributes: 11 + output attribute
  
   Note: several of the attributes may be correlated, thus it makes sense to apply some sort of
   feature selection.

7. Attribute information:

   For more information, read [Cortez et al., 2009].

   Input variables (based on physicochemical tests):
   1 - fixed acidity
   2 - volatile acidity
   3 - citric acid
   4 - residual sugar
   5 - chlorides
   6 - free sulfur dioxide
   7 - total sulfur dioxide
   8 - density
   9 - pH
   10 - sulphates
   11 - alcohol
   Output variable (based on sensory data): 
   12 - quality (score between 0 and 10)

8. Missing Attribute Values: None
